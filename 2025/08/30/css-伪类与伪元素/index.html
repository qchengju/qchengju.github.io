伪类和伪元素

一、核心区别：操作对象不同

维度 伪类（Pseudo-class） 伪元素（Pseudo-element）
操作对象 对已有元素在特定状态或场景下的样式修饰 创建不存在于 DOM 中的虚拟元素并设置样式
本质 元素的“状态/场景”（如 hover、first-child） 元素的“虚拟部分”（如首字母、前后内容）
语法 单冒号 :（如 :hover） 双冒号 ::（CSS3 规范，兼容兼容旧浏览器兼容单冒号）
数量限制 一个元素可同时应用多个伪类（如 :hover:active） 一个元素通常同时应用多个伪元素（仅能有一个）

注意： 关于上表中的数量限制， button:hover:active:focus 是伪类可以这样设置叠加状态，而伪元素在同一时刻只需对一个虚拟部分设置样式。

二、伪类（Pseudo-class）：修饰元素的特殊状态

伪类用于选中处于特定状态或满足特定条件的元素，这些元素在 DOM 中真实存在，只是样式会根据状态变化。

常见伪类及作用

  1. 状态伪类：根据元素交互状态修饰

    • :hover：鼠标悬停在元素上时
    • :active：元素被点击（按下未松开）时
    • :focus：元素获得焦点（如输入框被选中）时
    • :visited：链接被访问后

  2. 结构伪类：根据元素在 DOM 中的位置选中

    • :first-child：父元素的第一个子元素
    • :last-child：父元素的最后一个子元素
    • :nth-child(n)：父元素的第 n 个子元素（n 可写公式，如 2n 表示偶数位）
    • :only-child：父元素中唯一的子元素

  3. 其他伪类：

    • :empty：选中没有子元素的元素
    • :checked：选中被勾选的表单元素（如单选框、复选框）
    • :disabled：选中禁用状态的表单元素

三、伪元素（Pseudo-element）：创建虚拟元素

伪元素用于在元素的特定位置创建虚拟内容（这些内容不在 DOM 中），并为其设置样式。

常见伪元素及作用

  1. ::before：在元素内容前插入虚拟元素

  2. ::after：在元素内容后插入虚拟元素

    • 必须配合 content 属性使用（即使值为空 ""），常用于添加装饰性内容（如图标、分隔线）。

  3. ::first-letter：选中元素第一个字符（仅用于块级元素）

  4. ::first-line：选中元素第一行文本（仅用于块级元素）

  5. ::selection：选中用户鼠标选中的文本

四、总结：关键区分点

  1. 是否存在于 DOM：

    • 伪类修饰的是 DOM 中真实存在的元素（基于状态/位置）。
    • 伪元素创建的是不存在于 DOM 的虚拟元素（如 ::before 是额外生成的）。

  2. 用法场景：

    • 伪类：处理元素的交互状态（如 hover）或结构关系（如 first-child）。
    • 伪元素：处理元素的部分内容样式（如首字母）或添加装饰性内容（如 before/after）。

  3. 语法规范：

    • 伪类用单冒号 :（CSS3 兼容旧写法）。
    • 伪元素推荐用双冒号 ::（CSS3 新增，区别于伪类），但旧浏览器（如 IE8）仅支持单冒号。
