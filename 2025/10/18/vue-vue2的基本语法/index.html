vue2 基本语法

vue 的语法

介绍

vue虽然是javascript的一套框架，但是他依然有许多特有的语法，还是需要掌握的。对于引入vue源码的方式不做探讨，在正式的开发中，不会通过这种方式。无论是下载到本地还是直接用cdn建议在学习的时候这样用，开发中不会如此使用。

vue 指令

  1. 内容渲染指令
    v-text：
    用于绑定文本内容，当数据发生变化时，文本内容也会跟着改变，v-text 指令会覆盖元素内默认的值。
    v-html：
    用于绑定 HTML 内容，当数据发生变化时，HTML 内容也会跟着改变。使用此指令要注意安全，防止 XSS 攻击。
    v-pre：
    跳过当前节点的编译过程，将内容原封不动的插入到页面中。
    ：
    插值表达式，用于在模板中插入变量或者js表达式的值。可将内容渲染到指定位置，也不会覆盖元素里的默认值。

  2. 条件判断指令
    v-if：
    基于条件判断创建或者移除元素节点。
    v-show：
    基于条件控制元素的 display:none 样式属性。
    v-else/v-else-if：
    与v-if一起使用，辅助v-if进行判断渲染。需要紧接v-if指令。

  3. 事件绑定指令
    **v-on:**：
    用于绑定事件监听，可以简写为 @ 。

  4. 属性绑定指令
    **v-bind:**：
    可以动态设置 html 或者 vue 的标签属性。可以简写为 : 。

  5. 循环指令
    v-for：
    列表渲染指令，用来基于一个数组来循环渲染一个列表结构。
    v-for="(item, index) in list" 使用该指令时最好加上 key 属性，以便于 diff 算法正常运行。

  6. 表单双向绑定指令
    v-model：
    要注意的是，v-model 指令只能用于表单元素。
    作用：
    1、数据变化 => 视图自动更新
    2、视图变化 => 数据自动更新
    可绑定的元素示例：
    文本框 input:text => value
    文本域 textarea => value
    复选框 input:checkbox => checked
    单选框 input:radio => checked
    下拉菜单 select => value

  7. 插槽指令
    **v-slot:**：
    插槽指令，用于定义组件的模板。它是父组件使用的来填充子组件的语法，可以简写为 #.

  8. 与css结合的功能性指令
    v-cloak：
    这个指令保持在元素上直到关联实例结束编译。和 CSS 规则如 [v-cloak] { display: none } 一起用时，这个指令可以隐藏未编译的 Mustache 标签直到实例准备完毕。

  9. 指令修饰符

  • 针对于事件绑定 v-on

    1. Vue 事件修饰符

      修饰符 作用 示例
      .stop 阻止事件冒泡（等价于 event.stopPropagation()） <button @click.stop="handleClick">点击</button>
      .prevent 阻止默认行为（等价于 event.preventDefault()，如表单提交、链接跳转） <form @submit.prevent="handleSubmit">提交</form>
      .capture 事件捕获模式（从父元素到子元素触发） <div @click.capture="handleParent">父元素</div>
      .self 仅当事件触发在当前元素本身（非子元素）时执行 <div @click.self="handleSelf">点击自身才触发</div>
      .once 事件仅触发一次（触发后自动解绑） <button @click.once="handleOnce">仅触发一次</button>
      .passive 标记事件为”被动”（不调用 preventDefault()），优化滚动性能（如 touchmove） <div @touchmove.passive="handleScroll">滚动</div>
      .native 监听组件根元素的原生事件（用于自定义组件），vue3中已经移除 <MyComponent @click.native="handleNative">原生点击</MyComponent>

    2. Vue 按键修饰符

      修饰符 对应按键 示例
      .enter 回车键 <input @keyup.enter="handleEnter">回车触发</input>
      .tab Tab键 <input @keyup.tab="handleTab">Tab触发</input>
      .delete 删除/退格键 <input @keyup.delete="handleDelete">删除触发</input>
      .esc Esc键 <input @keyup.esc="handleEsc">Esc触发</input>
      .space 空格键 <input @keyup.space="handleSpace">空格触发</input>
      .up 上箭头键 <input @keyup.up="handleUp">上键触发</input>
      .down 下箭头键 <input @keyup.down="handleDown">下键触发</input>
      .left 左箭头键 <input @keyup.left="handleLeft">左键触发</input>
      .right 右箭头键 <input @keyup.right="handleRight">右键触发</input>

    3. Vue 系统修饰符

      修饰符 对应按键 示例
      .ctrl Ctrl键 <input @keydown.ctrl.s="handleSave">Ctrl+S 触发</input>
      .alt Alt键 <input @keydown.alt.s="handleAltSave">Alt+S 触发</input>
      .shift Shift键 <input @keydown.shift.s="handleShiftSave">Shift+S 触发</input>
      .meta Windows键/Command键 <input @keydown.meta.s="handleMetaSave">Win/Cmd+S 触发</input>
      .exact 精确匹配系统修饰符（仅按下指定修饰符时触发，不允许额外按键） <button @click.ctrl.exact="handleCtrl">仅 Ctrl+点击</button>

    4. Vue 鼠标按钮修饰符

      修饰符 对应鼠标按键 示例
      .left 鼠标左键 <div @mousedown.left="handleLeftClick">左键点击</div>
      .right 鼠标右键 <div @mousedown.right="handleRightClick">右键点击</div>
      .middle 鼠标中键 <div @mousedown.middle="handleMiddleClick">中键点击</div>

    这些修饰符可以组合使用，例如 @click.ctrl.shift="handleClick" 表示同时按下Ctrl和Shift键时点击才触发。

  • 针对于属性绑定 v-model

    1. .lazy 从 “输入时同步” 改为 “失去焦点 / 回车时同步”（减少数据更新频率）<input v-model.lazy="username">
    2. .number 将输入值自动转为 Number 类型（若输入非数字则保留字符串） <input v-model.number="age" type="number">
    3. .trim 自动去除输入值两端的空格（避免用户误输入空格） <input v-model.trim="password">

  • 针对于属性绑定 v-bind

    1. .prop 指定绑定的属性名称，如 v-bind:value="username" => v-bind.prop:value="username"
    2. .camel 将属性名称转换为驼峰命名法（如 data-username => dataUsername） <div v-bind.camel:data-username="username">
    3. .sync 创建一个 bidirectional 绑定，即 v-model 的语法糖，用于组件之间传递数据 <child-component v-model="username"></child-component>vue3中已经移除

!!!注意: 关于简写指令，一共有三个，建议使用直接简写，但是如果没有用简写，就这三个都尽量不要用简写，保持一致，这是一个规范问题并非语法限制。

computed计算属性

computed 语法

默认写法只能读取不能修改，如需修改，则需要使用setter方法，也就是第二种方式，写完整。

计算属性的特点

  1. computed属性的计算结果会被缓存，只有当依赖的属性值发生变化时才会重新计算。在遇到一些计算复杂的属性时，可以用计算属性来优化性能。
  2. computed属性的计算结果会自动添加到Vue实例的响应式系统中，所以当计算属性依赖的属性值发生变化时，计算属性也会自动更新。
  3. computed属性可以包含多个计算属性，多个计算属性之间可以相互依赖。
  4. computed 在v2中也是与 data 同级的属性。

watch侦听器

watch 语法

watch 侦听器的特点

  1. 只能侦听响应式属性，不能侦听普通属性,也就是 data 属性中声明的属性。
  2. immediate: 立即执行，是否在初始化时执行一次监听函数,默认为 false。
    deep: 是否深度监听,也就是针对对象嵌套属性进行深层监听，默认为 false。

v2的一些常用选项

vue2的整体语法是基于选项式api的。

el

el 是一个字符串，用于指定组件的挂载点，即在页面中指定的元素。只在 new Vue() 实例时才有效。
也就是一般都是挂到 root/app 上,由于 el 是替换原来的 dom 节点，因此一般不直接挂在 html/body 上。如官网给出的示例。

name

为组件指定一个名称，用于组件的注册和引用。

data

data 是一个对象，用于定义组件的初始数据。
data 可以是一个函数，返回一个对象，也可以是一个对象。
data就是声明组件的响应式数据。
在单文件组件中，data 必须是一个函数，以避免数据来源混乱。

props

props 可以是数组或对象，用于接收来自父组件的数据。

methods

methods 是一个对象，用于定义组件的方法。
在这个选项中，方法中的 this 指向组件实例。常用来处理一下组件的逻辑或者数据。

其他选项

有上面出现过的 computed、watch、还有那些生命周期的钩子函数均是选项。

vue2 中的 this 的使用

v2中几乎所有的数据和方法都是通过 this 来访问的，this 的作用域是当前组件实例，因此 this 可以访问当前组件实例的所有数据、方法、属性。
要访问组件资源或者访问 vue 内置的 api 都要用 this.
