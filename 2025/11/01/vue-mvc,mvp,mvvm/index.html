MVC -> MVP -> MVVM

发展与介绍

MVC（Model - View - Controller）

MVC基本概念

  • Model（模型）：负责处理数据和业务逻辑，比如数据的存储、读取、更新等操作。
  • View（视图）：负责展示数据和与用户进行交互，像界面布局、按钮、文本框等。
  • Controller（控制器）：作为 Model 和 View 之间的桥梁，接收用户的输入，调用 Model 进行相应的业务处理，并将处理结果传递给 View 进行展示。
  • react 框架使用 MVC 模式。

MVC工作流程

  1. 用户通过 View 发起请求。
  2. View 将请求传递给 Controller。
  3. Controller 调用 Model 进行业务处理。
  4. Model 处理完业务逻辑后，将结果返回给 Controller。
  5. Controller 将结果传递给 View 进行展示。

MVC优点

  • 职责分离：将数据处理、界面展示和用户交互逻辑分开，便于代码的维护和扩展。
  • 可测试性：Model 和 Controller 可以独立进行单元测试。

MVC缺点

  • View 和 Controller 耦合度高：View 和 Controller 之间的交互较为紧密，修改其中一个可能会影响到另一个。
  • 代码复杂度：随着项目规模的增大，Controller 可能会变得非常庞大，包含大量的业务逻辑。

MVP（Model - View - Presenter）

MVP基本概念

  • Model（模型）：与 MVC 中的 Model 类似，负责处理数据和业务逻辑。
  • View（视图）：负责展示数据和与用户进行交互，只负责界面的绘制和事件的触发，不包含任何业务逻辑。
  • Presenter（展示器）：作为 Model 和 View 之间的中介，接收 View 的请求，调用 Model 进行业务处理，并将处理结果传递给 View 进行展示。

MVP工作流程

  1. 用户通过 View 发起请求。
  2. View 将请求传递给 Presenter。
  3. Presenter 调用 Model 进行业务处理。
  4. Model 处理完业务逻辑后，将结果返回给 Presenter。
  5. Presenter 将结果传递给 View 进行展示。

MVP优点

  • 职责分离更彻底：View 只负责界面展示，Presenter 负责业务逻辑，两者的耦合度更低。
  • 可测试性强：Presenter 可以独立进行单元测试，不依赖于 View。

MVP缺点

  • 代码量增加：由于需要额外的 Presenter 层，代码量会有所增加。
  • 学习成本：对于初学者来说，理解 MVP 模式的概念和实现方式可能需要一定的时间。

MVVM（Model - View - ViewModel）

基本概念

  • Model（模型）：同样负责处理数据和业务逻辑。
  • View（视图）：负责展示数据和与用户进行交互，通过数据绑定机制与 ViewModel 进行关联。
  • ViewModel（视图模型）：作为 Model 和 View 之间的桥梁，将 Model 中的数据转换为 View 可以展示的格式，并处理 View 中的用户交互事件。

工作流程

  1. View 通过数据绑定机制从 ViewModel 中获取数据进行展示。
  2. 用户在 View 上进行交互操作，触发 ViewModel 中的相应事件。
  3. ViewModel 调用 Model 进行业务处理。
  4. Model 处理完业务逻辑后，将结果返回给 ViewModel。
  5. ViewModel 更新自身的数据，View 通过数据绑定机制自动更新界面展示。

优点

  • 数据绑定：通过数据绑定机制，View 和 ViewModel 之间的交互更加简洁，减少了手动更新界面的代码。
  • 可测试性高：ViewModel 可以独立进行单元测试，不依赖于 View。
  • 提高开发效率：开发人员可以专注于业务逻辑的实现，而不需要关注界面的更新。

缺点

  • 学习成本高：MVVM 模式引入了数据绑定和命令等概念，对于初学者来说学习成本较高。
  • 性能问题：数据绑定机制可能会带来一定的性能开销，尤其是在处理大量数据时。

总结

mvc 模式在前端的应用并不多，主要是后端的架构方式，当被逐渐优化 mvp 才渐渐被引入到前端的架构当中，后来又演变为 mvvm 在 Vue.js、Angular、React 这些框架中广泛使用

MVVM 模型在 vue 中

  1. 实现数据与视图的分离。
  2. 通过数据来驱动视图，开发者只需要关心数据变化，DOM操作被封装了
